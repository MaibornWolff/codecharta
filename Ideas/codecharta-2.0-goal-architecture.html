Nothing but the page knows more than one thing.

Zoom in: one page = the hub where everything meets

The key that makes it all line up: the node id cc.json 2.0 gives every node a stable id = sha-256(canonical path), 16 hex. The same id keys the structure tree, the metrics, and the edge endpoints — so lenses join without path-guessing, and highlight(id) / focusedNodeId resolve in any renderer. That's what enables the Graph→CodeMap "jump to this node."
Why this shape (and why your "edges in dependency" worry goes away) The CodeMap renderer never imports the dependency lens. The page reads dependency.edges() + structure.tree() + metrics + the chosen colorMetric, folds them into one { nodes, edges } model, and calls map.load(model). Edges are just a field in the model; the lens stays their owner; the renderer stays dumb. And each lens is a module that contains its own features (facade·models·components·services) over those repos — see lens-anatomy.html.
